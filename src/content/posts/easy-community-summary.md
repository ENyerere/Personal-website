易社区是我在校期间主导开发的一个社区平台项目,旨在为同学提供一个简洁、易用的交流空间。
本文记录项目的整体规划、技术选型与最终成果。

## 技术选型

| 模块     | 技术方案          | 说明                     |
| -------- | ----------------- | ------------------------ |
| 前端     | React + TypeScript | 组件化开发,类型安全     |
| 样式     | Tailwind CSS      | 原子化 CSS,开发效率高   |
| 状态管理 | Zustand           | 轻量、API 简洁           |
| 构建工具 | Vite              | 极速冷启动与 HMR         |

## 功能模块

项目主要包含以下功能模块:

- 用户认证(登录 / 注册)
- 文章发布与编辑
- 评论与点赞
- 标签分类与搜索

## 待办任务

以下是后续迭代计划:

- [x] 完成基础文章 CRUD
- [x] 接入主题切换(明 / 暗)
- [ ] 增加文章归档与标签云
- [ ] 优化移动端适配
- [ ] 接入评论通知系统

## 难点与思考

1. **状态共享**:早期使用 Context 导致频繁重渲染,后改用 Zustand 解决
2. **Markdown 渲染**:需考虑 XSS 防护与代码高亮的性能
3. **持久化**:通过 LocalStorage 缓存用户偏好与草稿

## 总结

易社区项目让我完整经历了一次从需求到落地的开发流程,
虽然仍有诸多不足,但每一步都让我对前端工程化有了更深的理解。
