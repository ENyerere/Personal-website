# Product

## Register

brand

## Users

中文读者,对前端开发、技术笔记、个人随笔感兴趣。他们通常在桌面端或移动端长时间停留阅读文章,期待安静的阅读环境,不希望被装饰元素干扰。读者可能是同行开发者、学生或技术爱好者,重视作者的独立声音而非通用教程站。

## Product Purpose

这是一个个人博客,用于发布技术笔记、项目总结与个人随笔。它提供清晰的文章归档与标签检索,展示作者身份与作品。成功标准:读者能专注阅读完整文章,并通过归档/标签自然发现更多内容;作者能以最低成本持续发布,无需维护复杂系统。

## Brand Personality

手稿感、档案化、精确。这个博客不是"网站",而是一叠被持续修订的手稿:每篇文章是一个有版本、有元数据、被认真归档的文档artifact。视觉语言来自纯文本世界——等宽字体、YAML front matter、细线、留白的纸面——而非来自 UI 组件库。技术感不靠霓虹与终端绿表达,而靠排版的纪律性表达:对齐、字距、行高、分隔线,全部有据可依。

3 词:manuscript, archival, precise

## Anti-references

- **圆角卡片网格**(旧版自身):卡片套卡片、大圆角、色块堆叠——手稿没有卡片,只有纸面与分隔线
- **SaaS 落地页**:满屏卡片、彩色渐变、营销话术——博客不卖东西
- **终端黑客风**:深色高对比、霓虹绿、扫描线——技术感由排版纪律表达,不靠 COS 终端
- **活泼装饰**:满屏 emoji、表情包、插画贴纸——装饰不应抢内容主角
- **论坛社区**:密集信息、复杂侧边栏、楼层式排版——博客不是讨论区

## Design Principles

1. **内容为王**:设计服务于阅读,排版、色彩、动画都为让读者专注于文字而存在,不抢内容主角
2. **文章即手稿**:每篇文章视为一份带元数据(YAML 式)、有版本历史的归档文档,而非信息流里的一条"内容"
3. **单色即态度**:不使用任何主题色相。墨(ink)与纸(paper)的对比驱动全部信息层级;链接靠下划线而非颜色识别
4. **细线取代容器**:用 1px hairline 分隔与组织内容,不用卡片背景、阴影、圆角堆叠来制造层次
5. **动画服务于流畅**:签名动效只有一种——标题的逐行揭示;其余动效一律克制(透明度、小位移),每个动画都应有明确目的
6. **静默的可访问性**:无障碍作为基线工程实践,不作为特殊处理或额外功能宣传
7. **组件优先**:优先复用成熟组件库(shadcn/ui)而非从零实现,但组件外观必须服从本设计系统

## Color System

纯单色双色系统,无语义主题色。明暗主题互为反转,共用同一套语义命名。

### 语义变量

| 变量 | 亮色 | 暗色 | 职责 |
|---|---|---|---|
| `paper` | `#fafafa` | `#0a0a0a` | 页面背景 |
| `ink` | `#0a0a0a` | `#fafafa` | 正文与标题文字 |
| `ink-2` | `#3d3d3d` | `#c4c4c4` | 次要文字(摘要、正文辅助) |
| `ink-3` | `#999999` | `#737373` | 静音文字(元数据、标签) |
| `line` | `#e5e5e5` | `#262626` | hairline 分隔线、边框 |
| `paper-2` | `#f2f2f2` | `#141414` | 代码块背景、引用块底色(极浅层次) |

### 规则

- **无 accent 色**:全站不存在"主题色"。可交互元素的识别方式是下划线、字重变化、bracket 符号,不是颜色
- **链接**:继承 `ink`,常显下划线(`text-underline-offset: 3px`),hover 时下划线加粗或位移;不靠变色
- **功能色唯一例外**:代码语法高亮可使用低饱和多色(保持单色阶梯推导,明度阶梯一致),`destructive` 保留单色红(仅危险操作)
- **暗色主题**:严格反转,`paper` 与 `ink` 互换,`line` 用 `#262626` 保持 hairline 可辨;代码块在明暗下均用深色 `paper-2` 暗色值
- **选中文本**:`::selection` 用 `ink` 底 `paper` 字(反转效果),不用彩色高亮
- **表面材质**:平面无阴影、无渐变、无玻璃拟态;层次仅由 hairline 与留白构成。允许极轻的纸面纹理(平铺微噪点,透明度 ≤3%),可后议

## Typography

双轨字体系统:**等宽驱动元数据与界面,人文无衬线驱动正文阅读**。中文渲染是硬约束。

### 字体栈

- **元数据/UI/标签/代码**:JetBrains Mono(已安装,启用 `calt`/`liga` OpenType 特性;后续可评估升级 Monaspace)。中文混排时等宽字体不覆盖 CJK,CJK 回落到系统黑体
- **正文/标题**:`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`(系统黑体,保证中文阅读质量与零加载成本)
- **不使用**:衬线中文正文(旧版已验证破坏长文阅读)、纯拉丁展示字体渲染中文标题

### 层级规则

- **标题**:系统黑体 + 字重区分,字距收紧 `letter-spacing: -0.015em`(大标题可到 `-0.03em`);中文标题不斜体(中文无真斜体)
- **标签体系**:拉丁小标签用 12px / 600 / uppercase / tracking 0.08em;中文标签用同字号 + 字重 500,不强行大写
- **YAML 元数据块**(签名元素):文章头部元信息以 front-matter 形式排布——标签名等宽、固定宽度(约 6em,`flex-shrink: 0`),值内联同行;如 `date: 2025-03-15` / `tags: [随笔, 个人]` / `words: 299`
- **正文**:18px 桌面 / 16px 移动,行高 1.75,measure 约 68 字符
- **代码**:JetBrains Mono,明暗主题下均置于深色 `paper-2` 背景

## Layout

告别卡片网格,改为**纸面 + hairline** 的编辑性布局。

### 页面骨架

- 页头:站点名(左)+ 导航(右),下方一条 hairline;sticky 时背景 `paper` 实色(不用毛玻璃)
- 页头与内容区之间可用一条 3px `ink` 粗规则线作为全站签名分隔(仅首页/归档等索引页,文章页省略)
- 容器最大宽度约 72rem;索引页内容区采用大留白(纵向 padding 6–8rem 量级)
- 页脚:**反转为墨色块**(`ink` 底 `paper` 字),超大联系排版收尾,形成明确的"卷末"感

### 文章列表(首页/归档)

- 无卡片。每篇文章是一行(或两行)条目,条目间以 hairline 分隔
- 条目结构:左侧等宽元数据栏(固定宽度,日期 `YYYY-MM-DD`)+ 标题(继承正文字号,不加粗过度)+ 静音摘要
- 允许轻微的错落节奏(奇偶条目缩进交替,幅度 ≤2rem)以呼应"散落手稿"的空间感,但不得破坏纵向扫描顺序
- hover:标题下划线出现,无位移、无缩放、无背景变化

### 文章页

- 单栏居中阅读栏(measure 约 68 字符),左右留白充足
- 文章头部:标题(逐行揭示动画)→ YAML 元数据块 → hairline → 正文
- 右栏 TOC(2xl 断点):hairline 风格,当前章节用字重 + 左侧短粗线指示,不用色块
- 上一篇/下一篇:hairline 分隔的文字链接行,不用卡片

### 移动端

- 单栏;元数据栏折行到标题上方;TOC 隐藏;页头折叠为菜单

## Motion

全站只有一种签名动效,其余一律克制。

### 签名动效:标题逐行揭示(line-reveal)

- 运行时用 JS 检测标题文本的自然分行(复制计算样式到隐藏测量元素,逐词重建检测换行),每行包裹 `.shift-line > span`
- 动画:`translateY(103%) → 0%`,时长 1400ms,缓动 `cubic-bezier(0.19, 1, 0.22, 1)`,逐行错峰(基础延迟 140ms + 每行递增 duration/10)
- 透明度补偿:每行 opacity 以 duration/3 线性淡入
- 触发时机:`document.fonts.ready` 之后的下一帧;文章标题入场用一次,不重复触发
- 用于:文章页 H1、首页站点签名区(如有);不用于正文任何标题

### 次级动效

- 列表条目入场:opacity + translateY(8px),错峰 `index × 100ms`,400ms,一次性
- 过滤切换:不匹配条目衰减至 `opacity: 0.3`(不移除、不重排),300ms
- 社交/页脚链接:下划线从左到右生长的 hover 动画(200ms)
- 滚动进度条、回到顶部:保留现有实现,样式改为 hairline/单色

### 降级

- `prefers-reduced-motion`:逐行揭示退化为直接显示(不拆分);所有过渡 ≤0.01ms;平滑滚动退化为瞬时跳转

## Components & Conventions

- **基础组件**:继续使用 shadcn/ui(Button/Badge/Sheet/NavigationMenu 等),但皮肤重写:无圆角(或小至 2px)、无阴影、hairline 边框、单色
- **过滤器约定**(标签/分类):文字按钮,激活态用 bracket 包裹——`> 前端 <`(`::before`/`::after` 注入),非激活为纯文字;不用 Badge 色块
- **展开/收起**(details/summary):自定义 `+` / `×` 标记(`::after`),展开态标记旋转或变色(单色内)
- **返回链接**:纯文字 `← 返回`,置于内容块上方,与内容左缘对齐(现行做法,保留)
- **版本链**(可选,配合功能候选 F4):文章元数据块附 git 风格短 hash,链接到该文历史版本
- **图标**:lucide 线性图标,单色,不放在彩色圆角容器里

## Accessibility

- WCAG AA:正文对比度 ≥4.5:1(ink/paper 天然满足),大字 ≥3:1
- 完整支持 `prefers-reduced-motion`(见 Motion 降级)
- 语义化 HTML、键盘可达、focus 态可见(`outline` 单色,不依赖颜色单独传达信息)
- 过滤器的激活态除 bracket 外同步 `aria-pressed`;TOC 当前章节 `aria-current`

## Migration Notes(从现风格迁移的影响面)

- `globals.css`:OKLCH `--hue` 体系 → ink/paper 语义变量(HEX 或 OKLCH 均可,保持 `<alpha-value>` 解析方式);`.card-base`/`.btn-regular` 弃用
- `tailwind.config.js`:色彩映射改为语义变量;圆角语言重定义;字体栈更新
- 组件:`PostCard` → 行式条目;`SideBar` Profile/标签卡 → hairline 区块或并入页头/页脚;`TagCloud` Badge → bracket 过滤器;`TOC` 样式重写(逻辑保留)
- 动效:GSAP + SplitText 移除(包体 −70KB 量级),换自研 line-reveal(~40 行 JS);错峰入场 CSS 重写
- 页面:首页/归档/标签/关于按 Layout 章节重排;文章页头部改为 YAML 元数据块
- 保留不动:路由结构、postStore 与选择器、useTheme、TOC 逻辑、reduced-motion 基建、SPA 404 回退
